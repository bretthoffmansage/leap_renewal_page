# DESIGN_DIRECTION_BRIEF_TEMPLATE

## Purpose

This document defines the art direction and UX posture for Phase F (High-End Design Pass).

It should convert a functional but developer-centric interface into a polished, cohesive product experience without breaking functionality.

## Functional preservation lock

Before design changes, explicitly preserve:

- routes/pages
- core workflows
- forms and form behavior
- search/filter behavior
- permissions/access controls
- loading/error behavior
- data/API behavior and business logic
- dense/raw data access pathways

## Dense/raw data handling strategy

If content is messy but valuable, preserve it through better patterns:

- tabs
- collapsible sections
- drawers
- detail panels
- advanced/raw data areas

Do not delete valuable information only because it is visually dense.

---

## Out of scope

- speculative feature expansion
- workflow deletion
- gimmicky animation
- readability-sacrificing decoration
